## 기능 요구사항
간단한? 문자열 계산기를 구현한다.  
  
- 사용자에게 데이터를 1회 입력받는다. 데이터는 실제 기본적인 사칙 연산을 표현한 문자열을 입력받으며 적용해야 할 각 규칙들은 다음과 같다.
- `+`, `-` : 각각 덧셈과 뺄셈의 의미를 가지고 있다
- `*`, `/` : 각각 곱셈과 나눗셈의 의미를 가지고 있다.
- `()` : 이 안에 있는 연산을 최우선적으로 진행한다.
- 우선순위가 없을 경우 왼쪽에서 오른쪽으로 계산을 먼저 진행한다.
- 예를 들어 `1+2*3`의 경우에는 `7`을 반환해야 하고 `(1+2)*3`은 `9`를 반환해야 한다.
- 사전에 주어지는 계층 조건을 만족해서 기능을 구현해야 한다.
  
### 입출력 요구 사항
#### 입력
- 사칙 연산 기호가 포함된 문자열을 1회 입력받는다.  
``` console
# case 1
1+2*3
```
- 만약 소괄호가 포함되어 있는 연산 부분이 있을 경우 그 부분을 최 우선적으로 계산해야 한다.  
``` console
# case 2
(1+2)*3
```
#### 출력
- 연산 결과를 출력한다.  
``` console
# case 1
결과 : 7

# case 2
결과 : 9
```
#### 실행 결과 예시
``` console
# case 1
연산할 문자열을 입력해 주세요.
1+2*3
결과 : 7

#case 2
연산할 문자열을 입력해 주세요.
(1+2)*3
결과 : 9
```
## 프로그래밍 요구 사항 1
- JDK 21 버전에서 실행 가능해야 한다.
- 프로그램 실행의 시작점은 `Application`의 `main()`이다.
- `build.gradle`파일은 변경할 수 없으며, **제공된 라이브러리 이외의 외부 라이브러리는 사용하지 않는다.**
- 프로그램 종료 시 `System.exit()`를 호출하지 않는다.
- 자바 코드 컨벤션을 지키면서 프로그래밍한다.
	- 기본적으로 [Java Style Guide](https://github.com/woowacourse/woowacourse-docs/blob/main/styleguide/java)를 원칙으로 한다.

## 프로그래밍 요구 사항 2
- 어플리케이션은 반드시 각 3개의 계층 클래스 객체를 구현해야 한다
- InputView, OutputView : 각각 입력과 출력에만 집중해야 하며 입력 뷰는 입력을 받아서 리턴하여야 하고 출력 뷰는 입력을 받아서 출력해야 한다.
- Controller : 제어를 하는 역할이다. 뷰에서 데이터를 받고 도메인을 활용해서 해당 문제 조건을 처리할 수 있는 비즈니스 로직을 수행한 후 완료한 결과를 다시 뷰에게 전달한다. 컨트롤러는 뷰를 제외한 필드를 가지는것을 금지한다.
- Domain : 실제 데이터를 가질 수 있는 객체이며, 생성해서 사용하는 것을 허가한다. 이 내용은 자유롭게 구상해보는것을 허가한다.

## 프로그래밍 요구 사항 3
- 잘못된 입력에 대해서는 IlligalArgumentException을 떨궈야 한다. 예외를 떨구는 지점은 자유롭게 선택한다. 예외에 적절한 메시지도 포함해 보는것을 목적으로 하자.
